javascript 创建class

前言

  OOP是什么，OOP的好处是什么本文不做赘述。本文只讨论怎么样使用javascript定义class和对象的几种方法以及优缺点。

字面量方式

  这种方式的优点是可以快速得到一个我们想要的对象，特别是在将对象作为参数或返回值时，大量使用。但他的缺点非常明显，第一、无法传参；第二、无法使用typeof操作符获得对象类型。

工厂模式

  工厂模式抽象了具体创建对象的过程细节，可以根据参数来创建不同对象。工厂方法的缺点是无法知道对象的类型(无法使用typeof操作符)，这点与字面量方式创建对象的方式相似。

构造函数模式

  使用构造函数模式相当于定了class，可使用new操作符来创建该class的对象。使用new操作符创建对象有4步骤：

  1. 创建一个新对象
  2. 将构造函数的作用域赋值给新对象 (也就是将this指向了新对象)
  3. 执行构造函数中的代码 (添加对象属性和初始化对象属性值)
  4. 返回新对象

每个对象都有一个constructor属性，指向对象的构造函数。构造函数与普通函数的区别就在于它的调用方式。任何函数只要同过new操作符来调用，该函数就是构造函数；否则它就是普通函数。
  构造函数的最大问题就在于对象方法(method)的定义。调用相同构造函数创建的不同对象中的方法都是不同的Fucntion实例。

一种解决办法是将方法指向外部函数，但这种方法会引起全局函数大爆炸，以及代码混乱且极不优雅，不推荐，就不上代码了。
