git pull 致命错误

  在GitHub上新建仓库(repository)时，时常建议你不要勾选 Initialize this repository with a README 选修。其原因是，如果你的项目已在本地仓库中存在了，此时需要使用GitHub托管。如果你勾选了上述的选修，就等于在新建的仓库提交了一个版本。此时新建的GitHub远程仓库其实不是一个空仓库。此时使用 push 命令会被告知无法提交，git系统建议先进行 pull 操作。运气不错的话，进行一次 pull 操作，系统提示进行 merge 完事。运气不好，就会遇到提示致命错误。

这个错误产生的原因是，git系统认为你的已有本地仓库与远程GitHub仓库其实是无关的两个独立仓库。对于这两个仓库，git拒绝合并(merge)。我们可以选择强制合并解决此问题，即使用 –allow-unrelated-histories 参数进行 pull
。
