前言

原型与原型链

前言

  我们理解原型对象概念之前，先要搞清楚函数对象,原型对象,实例对象这几个对象之间的关系。
  函数对象，在我们声明函数以后，代码解析完成，解释器即会为每一个函数(或函数表达式)生成函数对象。并为函数对象创建一个prototype属性，这个属性是一个指向函数原型对象的指针。
  函数原型对象(prototype)，解析器在创建函数对象的同时会创建函数默认的原型对象。原型对象与所修饰的函数对象形成一对一的相互引用关系。原型对象默认包含一个[constructor]属性，这个属性是一个指向原型所修饰的函数对象的指针。
  实例对象，使用new操作符调用构造函数等OOP方式生成的类的实例。实例对象中包含一个[[Prototype]]指针，它指向构造函数的原型对象。某一个类的若干个的实例对象的[[Prototype]]指针指向同一个构造函数的原型对象。实例对象与函数对象之间相互没有引用关系。
  由此可以看到，函数对象与原型对象成对出现，实例对象只与原型对象之间有引用关系。

原型

  原型对象本质上就是普通的javascript对象，只是他与函数对象之间产生了特殊的引用关系。我们可以自己任意定义一个对象，然后使之与函数对象产生特定的引用的关系(即改变函数对象的prototype指针以及原型对象的constructor指针的指向)，那么函数的原型对象即被修改为我们定义的对象。例如：

javascript 创建class

前言

  OOP是什么，OOP的好处是什么本文不做赘述。本文只讨论怎么样使用javascript定义class和对象的几种方法以及优缺点。

字面量方式

  这种方式的优点是可以快速得到一个我们想要的对象，特别是在将对象作为参数或返回值时，大量使用。但他的缺点非常明显，第一、无法传参；第二、无法使用typeof操作符获得对象类型。

工厂模式

  工厂模式抽象了具体创建对象的过程细节，可以根据参数来创建不同对象。工厂方法的缺点是无法知道对象的类型(无法使用typeof操作符)，这点与字面量方式创建对象的方式相似。

构造函数模式

  使用构造函数模式相当于定了class，可使用new操作符来创建该class的对象。使用new操作符创建对象有4步骤：

  1. 创建一个新对象
  2. 将构造函数的作用域赋值给新对象 (也就是将this指向了新对象)
  3. 执行构造函数中的代码 (添加对象属性和初始化对象属性值)
  4. 返回新对象

每个对象都有一个constructor属性，指向对象的构造函数。构造函数与普通函数的区别就在于它的调用方式。任何函数只要同过new操作符来调用，该函数就是构造函数；否则它就是普通函数。
  构造函数的最大问题就在于对象方法(method)的定义。调用相同构造函数创建的不同对象中的方法都是不同的Fucntion实例。

一种解决办法是将方法指向外部函数，但这种方法会引起全局函数大爆炸，以及代码混乱且极不优雅，不推荐，就不上代码了。

原型模式&构造函数

  javascript原型(prototype)是个重要概念，这里不做讨论。感兴趣可一个看这篇文章。

javascript 执行环境

先说下javascript的函数

  与Java不同，javascript函数会被当做作用域中的变量。且在代码被读取、初始化时就会生成Function类型的对象变量，不管其有没有被调用都会生成。函数对象生成后，其内部会包含一个初始的作用域链对象。

什么是执行环境

  执行环境(execution context)是javascript中最重要的一个概念。它定义了函数执行时变量的访问权限和查找顺序，理解执行环境对彻底理解闭包至关重要。
  执行环境是一个对象，后台通过对此对象相关的变量对象和作用域链的配置、访问来完成执行环境的实际工作。每一个函数都有自己的执行环境，其概念类似于方法执行栈。函数执行时保存当前(函数)执行环境的变量、函数(javascript中函数也是变量)，执行结束后销毁这些变量、函数。执行环境对象也在函数执行完后被销毁。

变量对象(variable object)

  它是一个javascript对象，某个执行环境定义的所有变量、对象都保存在变量对象中。除全局作用域外，函数执行环境的对象变量只在函数执行过程中存在，函数执行完即被销毁。变量对象中包含arguments对象，而全局作用域没有此对象。与变量对象相对应的是活动对象。

活动对象(acrivation object)

  正在执行中的函数的变量对象叫做活动对象。活动对象一开始只有arguments和this两个对象变量，函数中的变量、函数在函数执行时，被初始化并添加到活动对象中。

作用域链(scope chain)

  作用域链控制着变量查找的顺序。作用域链在函数创建时(产生Function对象)，被保存在函数(Function)对象内部的[[Scope]]对象中。它内部是一系列指针，指向对应的对象变量。值得注意的是，在函数被创建并没有被调用时，[[Scope]]中仅有对全局对象变量的指向。在函数被调用时，系统会生成一个执行环境对象，然后通过复制函数对象中的[[Scope]]对象，构建执行环境对象的作用域链。而后再生成执行环境的活动对象，并将作用域链的第一个指针指向活动对象，使之成为作用域链中第一个变量对象(作用域链的最前面)。函数执行完，作用域链会随着函数的执行环境对象一并被销毁(特殊情况除外，比如闭包)。

什么是属性特征？

  我们创建js对象时都会为对象设定属性(property)，而由于javascript特性，属性(property)可以自由添加，删除，类型修改。javascript属性特征就是用来描述以及设定这些属性是否拥有这些特性，即属性是否可以自由添加，删除，类型修改。
  先来查看一下属性的属性特征吧。使用Object.getOwnPropertyDescriptor函数可以查看对象属性的特征。

再来看看访问器(getter/setter)的属性特征

  下面我在就来详细的讨论下，这些属性特征的含义。javascript属性分为两种，即：数据属性和访问器属性

  1. 数据属性
    对象包含的变量、函数都是对象数据属性。 数据属性包含以下特征：
  • [[Configurable]]
    字面意思表示用来修饰属性是否可以配置，也就是属性是否可以删除，属性的其它属性特征是否可以重新定义，属性值是否可以重新赋值。它的值为false表示不可以，true表示可以，默认值为true。它的值为true的话，属性的其它属性特征的值就显得不那么重要了，因为可以根据需要重新配置属性特征。
  • [[Enumerable]]
    是否可以遍历。定义了在对对象使用for-in循环时属性是否可以显示。默认值是true
  • [[Writable]]
    是否可以修改属性的值。值为false的话，基本可以认为改属性为常量。默认值是true
  • [[Value]]
    属性值。默认值是undefined
  1. 访问器属性
    他们不包含数据值，他们是一对(也可不成对)包含getter和setter的函数(此处要区别与普通函数)。访问器属性，除对象字面量以外，其他的生成对象的方式都不支持直接定义，必须使用Object.defineProperty函数来定义。例如：

访问器属性包含以下特征：

  • [[Enumerable]]
    同数据属性
  • [[Configurable]]
    同数据属性
  • [[Get]]
    读取属性时调用的函数，默认值是undefined
  • [[Set]]
    在写入属性时调用的函数，默认值是undefined

修改对象属性默认特征

修改属性默认特征必须使用ES5提供的Object.defineProperty或Object.defineProperties函数。
